GIT-Clone

Allgemeine Beschreibung zu diesem Befehl

Ein schon (auf GitHub) existierendes Repository kann durch git clone lokal heruntergeladen werden. Dabei wird ein Klon, beziehungsweise eine exakte Kopie erstellt.
Alle Dateien, branches und commits bleiben dabei enthalten. Mit dem geklonten Repository kann dann lokal gearbeitet werden.

Wann GIT-Clone verwenden?

Ähnlich wie git init ist auch git clone ein Befehl, der in der Regel nur einmal ganz am Anfang des Projektes ausgeführt wird.
Zum Start benötigt man einmal alle schon erstellten Dateien und Ergebnisse.
Im Laufe des Projektes würde es allerdings keinen Sinn machen, das Repository noch einmal zu klonen, da man durch regelmäßige push und pull Befehle immer auf dem aktuellen Stand des Projektes sein sollte.
